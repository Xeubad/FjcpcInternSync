## 跨项目通用经验（占位说明）

如果发现"换个项目也成立"的经验，**不要写在这里**——直接追加到 `~/.claude/memory/lessons.md`。

判断标准：**"这条经验脱离当前仓库后还成立吗？"**

- ✓ 成立 → 写全局 `~/.claude/memory/lessons.md`
- ✗ 不成立 → 写本文件

跨项目通用的经验示例（这些**应该**写到全局而非本文件）：

- "TypeScript strict 模式下，Object.assign 的返回类型推断会丢"
- "npm 在 monorepo 里跑 install 比 pnpm 慢 3x"
- "Postgres 的 jsonb 字段加 GIN 索引能让查询从 2s 降到 30ms"
- "测试夹具用 factory-bot 比 fixture 好维护"

项目专属经验的示例（这些**才**写本文件）：

- "本项目用户表的 `email` 字段在 v2 schema 是必填，v1 是可选——迁移逻辑见 `migrations/0042`"
- "本项目使用 Stripe 测试模式时，必须用 4242 4242 4242 4242，团队规约"
- "本项目 CI 在 `feature/*` 分支默认跳过 e2e 测试以省时间"

## 写入纪律

- 写之前问自己：**"这条经验脱离当前仓库后还成立吗？"** 成立 → 全局；不成立 → 本文件
- 每条必须能回答"下次撞到时怎么办"，否则不写
- 用户同一类纠正出现 2 次以上 → 必须沉淀
- 过期 / 被新规则取代时划掉但保留历史，不删除
- 本文件由 Stop hook 在每次会话结束时提醒"是否需要追加新经验"
