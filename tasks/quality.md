# 质量快照

> 用途：定期评估**本项目**整体质量是变强还是变弱。
> 频率：每个大版本完成时 / 每月一次 / 大重构后。
> 由 `/quality-snapshot` slash 命令触发，或 Stop hook 在重要里程碑后提醒。

---

## 评分维度速查

每个维度用四级评：**优 / 良 / 中 / 差**

| 维度 | 含义 |
|---|---|
| 验证状态 | 是否有可运行的自动化验证（测试、lint、类型检查） |
| 可读性 | 新人接手时多快能看懂结构与命名 |
| 稳定性 | 改一处会不会牵连一片 |
| 关键差距 | 当前最大的痛点（必须可执行，不能是"待观察"） |

## 维护原则

- 每次大版本完成时新增一节快照
- 风险条目必须可执行（"测试覆盖低" → 不可执行；"补 OAuth 回调测试" → 可执行）
- 历史快照保留作为演进档案，不删
- 与 `progress.md` 的区别：progress 记"做了什么"，quality 记"做完之后整体好不好"
